34 // We override many read-methods because PlasticCloneConn will use
35 // originalConn's values. communicateInitInfo will check if the associated
36 // parameters exist in params for theCloneKernelConn group, and whether they
37 // are consistent with the originalConn parameters.
38 // If consistent, issue a warning that the param is unnecessary and continue.
39 // If inconsistent, issue an error and quit.
40 // We can't do that in the read-method because we can't be sure originalConn
41 // has set its own parameter yet (or even if it's been instantiated),
42 // and in theory originalConn could be a subclass that determines
43 // the parameter some way other than reading its own parameter
44 // group's param directly.
