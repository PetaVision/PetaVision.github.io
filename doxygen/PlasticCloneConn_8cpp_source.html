30 // We override many read-methods because PlasticCloneConn will use
31 // originalConn's values. communicateInitInfo will check if the associated
32 // parameters exist in params for theCloneKernelConn group, and whether they
33 // are consistent with the originalConn parameters.
34 // If consistent, issue a warning that the param is unnecessary and continue.
35 // If inconsistent, issue an error and quit.
36 // We can't do that in the read-method because we can't be sure originalConn
37 // has set its own parameter yet (or even if it's been instantiated),
38 // and in theory originalConn could be a subclass that determines
39 // the parameter some way other than reading its own parameter
40 // group's param directly.
